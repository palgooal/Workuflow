# دعم العملات المتعددة — دراهم

> تاريخ الإنشاء: 29 مايو 2026  
> الإصدار: 1.0.0

---

## المشكلة

النظام يدعم معاملات وفواتير بعملات مختلفة (ILS، USD، EUR…)، لكن صفحات الملخص المالي كانت تجمع المبالغ بدون اعتبار العملة — فتظهر أرقام مضللة (مثل جمع 5,000 ₪ + $1,200 = 6,200 بلا معنى).

## القرار التصميمي

**لا تحويل — عرض منفصل.** بدلاً من ربط API أسعار صرف خارجي أو تخزين أسعار صرف يدوية، يعرض النظام المبالغ مجمّعة حسب العملة مع تحذير واضح للمستخدم.

هذا القرار:
- أمين (لا تقريب بأسعار صرف قديمة)
- بسيط التنفيذ والصيانة
- يترك للمستخدم حرية التفسير

## الملفات المعدَّلة

### Services

**`TransactionService::getSummary()`**
```php
// يُرجع الآن:
[
    'by_currency' => [
        'ILS' => ['income' => 5000, 'expenses' => 1200, 'net' => 3800],
        'USD' => ['income' => 800,  'expenses' => 200,  'net' => 600],
    ],
    'multi_currency' => true,
    // للتوافق مع الكود القديم (أول عملة):
    'income'   => 5000,
    'expenses' => 1200,
    'net'      => 3800,
    'count'    => 45,
]
```
- الفلاتر (type, category, project, date) مطبّقة قبل التجميع

**`ProjectFinancialService::getSummary()`**
```php
// يُرجع الآن:
[
    'by_currency'      => [...],   // per-currency
    'multi_currency'   => bool,
    'project_currency' => 'ILS',   // عملة المشروع الأساسية
    // قيمة العقد والميزانية: تُحسَب بعملة المشروع فقط
    'contract_collected' => 75.0,
    'budget_used_percent' => 60.0,
    // ...
]
```

**`ProjectFinancialService::getPortfolioSummary()`**
- يجمع معاملات جميع المشاريع per-currency
- يُرجع `by_currency` + `multi_currency`

### Views

| الصفحة | عملة واحدة | عملات متعددة |
|--------|-----------|-------------|
| `/transactions` | 3 بطاقات (دخل/مصروف/صافي) | جدول صف لكل عملة |
| `/projects` | 4 بطاقات KPI مع اسم العملة | جدول 4 أعمدة per-currency |
| `/projects/{id}` | 4 بطاقات KPI مع اسم العملة | جدول 5 أعمدة (+ هامش الربح) |
| `/clients/{id}` | 3 بطاقات (إيراد/مدفوع/مستحق) | سطر لكل عملة في كل بطاقة |

### البانر التحذيري

يظهر في جميع الصفحات عند وجود عملات متعددة:
```html
<div class="bg-amber-50 border border-amber-200 ...">
    المبالغ معروضة منفصلة لكل عملة — لا يتم دمجها تفادياً للخطأ في الحساب.
</div>
```

## ملاحظات خاصة بالمشاريع

- **قيمة العقد** (`contract_value`) مخزّنة بعملة المشروع (`$project->currency`)
- **الميزانية** (`expense_budget`) أيضاً بعملة المشروع
- عند وجود عملات متعددة، تُحسَب نسبة الاستلام والميزانية من معاملات عملة المشروع فقط
- البانر يوضح ذلك: "مقارنة قيمة العقد والميزانية تعتمد على عملة المشروع (ILS) فقط"

## إضافات مستقبلية

### أسعار الصرف اليدوية
المستخدم يدخل في الإعدادات: 1 USD = 3.7 ILS
النظام يستخدمها لتوليد ملخص موحّد بالعملة الأساسية بجانب الجدول المنفصل.

### ربط API أسعار الصرف
استخدام API مجاني (مثل exchangerate-api.com) لجلب أسعار اليوم تلقائياً وتحديثها يومياً عبر Scheduled Task.

### العملة الأساسية
إضافة `base_currency` لإعدادات المستخدم (`users.base_currency`)، وعرض إجمالي محوّل بجانب الجدول المنفصل.
